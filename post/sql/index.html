SQL基础

Join连接

  • INNER JOIN：如果表中有至少一个匹配，则返回行
  • LEFT JOIN：即使右表中没有匹配，也从左表返回所有的行
  • RIGHT JOIN：即使左表中没有匹配，也从右表返回所有的行
  • FULL JOIN：只要其中一个表中存在匹配，则返回行
  • SELF JOIN：用于将表连接到自己，就好像该表是两个表一样，临时重命名了SQL语句中的至少一个表
  • CARTESIAN JOIN：从两个或多个连接表返回记录集的笛卡儿积

SQL高级

锁

  • 悲观锁 – 当事务A对某行数据加了锁，只有当事务A把锁释放后，其它事务才能执行与该锁冲突的操作。
  • 乐观锁 – 假设多用户并发的事务在处理时互不影响，各事务能在不产生锁的情况下进行处理各自影响的那部分数据，在提交数据更新前检查该事务读取数据后其它事务有没有修改数据，如果有，当前事务进行回滚。

事务

原子性是基础，隔离性是手段，一致性和持久性是目标。

  • Atomic（原子性）– 事务中的操作被视为一个逻辑单元，要么全部成功，要么全部失败。
  • Consistency（一致性）– 只有合法的数据可以被写入数据库，否则事务应该将其回滚到最初状态。
  • Isolation（隔离性）– 事务允许多个用户对同一个数据进行并发访问，而不破坏数据的正确性和完整性。同时，并行事务的修改必须与其他的并行事务的修改相互独立。
  • Durability（持久性）– 事务完成后之，它对数据库的影响是永久的。即使出现致命的系统故障也将一直保存。

隔离级别由低到高：Read Uncommitted < Read Committed < Repeatable Read < Serializable

  • 脏读 – 事务A读取了事务B未提交的数据，并在这个数据基础上又做了其他操作。
    • 一般是事务未隔离造成的
  • 不可重复读 – 事务A读取了事务B已提交的更改数据。
    • 重复读取的数据不一样
  • 幻读 – 事务A读取了事务B已提交的新增数据。
    • 重复读取的数据不一样，可能读取了其他事务的新增数据。

方法一：
select a.id,a.SName,a.ClsNo,a.Score
from Table1 a left join Table1 b on a.ClsNo=b.ClsNo 
and a.Score<b.Score
group by a.id,a.SName,a.ClsNo,a.Score 
having count (b.id)<2
order by a.ClsNo,a.Score desc
方法二：
select *
from Table1 a
where (select count (*) from Table1 b where b.ClsNo=a.ClsNo and b.Score>a.Score)<2
order by a.ClsNo,a.Score desc
方法三：
select *
from Table1 a
where id in (select  id from Table1 b where b.ClsNo=a.ClsNo order by b.Score desc limit 2 )
order by a.ClsNo,a.Score desc
