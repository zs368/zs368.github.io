laradock

  1. 修改 .env 文件如下

    1
2
3

    WORKSPACE_INSTALL_XDEBUG=true  
   
PHP_FPM_INSTALL_XDEBUG=true

  2. 修改 workspace/xdebug.ini 和 php-fpm/xdebug.ini 中的两行如下

    1
2
3

    xdebug.remote_host=host.docker.internal
   
xdebug.remote_enable=1

  3. docker build workspace php-fpm

PHPstorm

  • WEB 调试

    1. settings > languages & frameworks > PHP > Servers

      1. 新增名为 laradock下.env 文件中servername的名字（默认laradock）
      2. host 名字为该项目域名
      3. 目录需映射到docker中对应项目地址

      修改完可以去同级 Debug 下Validate下看看是否可以（注意path需到public目录下）

    2. run > edit configurations

      1. 新增 PHP Remote Debug
      2. name随便（debug），server为刚才配置的名字（laradock）
      3. IDE key 为 PHPSTORM

      这里也可以 Validate 下

    3. run > 选取刚才配置的（debug） > start listening for php debug connections

  • CLI 调试

    1. settings > languages & frameworks > PHP
      1. 选择 CLI Interpreter
      2. 没有对应的就新增，从docker中选择workspace
      3. 选择 PHP executable，测试OK
    2. run > edit configurations
      1. 新增 PHP Script
      2. name随便（cli），file 选择为根目录下的 artisan，参数选择要测试的命令
      3. Interpreter 选择刚才新增的
    3. run > 选取刚才配置的（cli）> 点击 Debug
